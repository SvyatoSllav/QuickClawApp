## 1. Текущие версии и обновления

### Flutter

- **Текущая стабильная версия:** Flutter 3.38.6 (конец 2025). За 2025 год вышло 4 стабильных релиза (3.29 - 3.38).
- **Hot Reload для веба:** Stateful hot reload для веб-приложений появился как экспериментальный в 3.32, стал стабильным в 3.35.
- **WebAssembly:** Стабильная поддержка в 3.38, но остаётся opt-in через флаг `--wasm`.
- **AI-интеграция:** GenUI SDK (альфа) для LLM-генерации UI; Flutter AI Toolkit v1.0 вышел в декабре 2025.
- **Ожидается:** Flutter 3.41 и Dart 3.11 в феврале 2026. Flutter 4.0 прогнозируется на середину 2026 с модульным ядром, оптимизациями Impeller и улучшенной поддержкой десктопа.

### React Native

- **Текущая стабильная версия:** React Native 0.83 (декабрь 2025) с нулевым количеством breaking changes и интеграцией React 19.2.
- За 2025 год вышло 7 мажорных релизов — один из самых продуктивных годов за всю историю фреймворка.
- **New Architecture по умолчанию:** После 6 лет разработки новая архитектура (TurboModules, Fabric Renderer, JSI) стала дефолтной в 0.76 (сентябрь 2025). Старый bridge-механизм объявлен deprecated.
- **Улучшения CSS:** Добавлены `display: contents`, `boxSizing`, `mixBlendMode`, `outline`; поддержка edge-to-edge для Android 15.
- **Ожидается:** React Native 1.0 анонсирован на React Conf 2025 как «на горизонте» после 10 лет разработки.

## 2. Производительность

| Метрика | Flutter | React Native |
|---------|---------|-------------|
| Частота кадров | 60-120 FPS стабильно | 95% от нативной производительности |
| Загрузка CPU | ~43.42% | ~52.92% |
| Время первого кадра | < 50 мс (самый быстрый) | < 50 мс (самый стабильный) |
| Потребление памяти | Умеренное, стабильное | Растёт, особенно на iOS |
| Сложные анимации | Превосходит на 30-50% | Уступает при тяжёлых анимациях |

**Ключевые факты:**

- Движок **Impeller** во Flutter улучшает рендеринг кадров на ~30% по сравнению со старым Skia (по бенчмаркам Alibaba и Google).
- Импеллер полностью устраняет shader compilation jank — «заикания» при первой отрисовке сложных шейдеров.
- React Native с новой архитектурой (**JSI** вместо bridge) значительно сократил задержки. По данным Natively (2025), RN достигает 95% нативной производительности рендеринга в стандартных UI-сценариях.
- Flutter стабильно выигрывает в тяжёлых анимациях, вложенных списках и кастомных UI-компонентах.

## 7. Архитектурные изменения

### Flutter: Impeller Engine

- **Impeller — дефолтный движок рендеринга** на iOS и Android в 2025.
- На iOS Skia полностью удалена.
- На Android Impeller работает по умолчанию на API 29+ (Flutter 3.38 deprecated opt-out).
- Веб-рендеринг всё ещё использует Skia (Impeller для веба пока недоступен).
- Преимущества: предкомпилированные шейдеры (устранение jank), ускоренный рендеринг текста, эффективные эффекты размытия, улучшенное управление памятью, более быстрый запуск движка.
- **Impeller 2.0** ожидается с Flutter 4.0 (середина 2026).

### React Native: New Architecture

- Три столпа: **Fabric** (новая система рендеринга), **TurboModules** (замена Legacy Native Modules), **JSI** (JavaScript Interface, замена bridge).
- Стала дефолтной в RN 0.76 (сентябрь 2025); старая архитектура deprecated.
- **Hermes** — дефолтный JS-движок: более быстрый запуск, меньший размер бандла, улучшенная эффективность памяти по сравнению с JavaScriptCore.
- Bridge полностью устранён — вся коммуникация через C++ JSI слой.
- Миграция настоятельно рекомендуется для долгосрочной стабильности проектов.

## 8. Поддержка десктопа и веба

### Десктоп

| Платформа | Flutter | React Native |
|-----------|---------|-------------|
| Windows | Стабильная (first-party) | Через RN for Windows (Microsoft) |
| macOS | Стабильная (first-party) | Через RN for macOS (Microsoft) |
| Linux | Стабильная (first-party) | Нет официальной поддержки |
| Зрелость | Единый рендеринг-стек | Менее зрелая, community-maintained |

Flutter значительно опережает по десктоп-поддержке: единый стек рендеринга обеспечивает консистентный опыт на всех платформах.

### Веб

| Аспект | Flutter | React Native |
|--------|---------|-------------|
| Подход | Canvas/WebGL/WASM | DOM-рендеринг (через React Native Web) |
| Зрелость | Стабильный, но лучше для SPA/дашбордов | Более зрелый для традиционного веб-контента |
| SEO | Проблематичен для контентных сайтов | Лучше (DOM-based) |
| WebAssembly | Стабильный (opt-in) | Не применимо |
| Hot Reload | Стабильный с 3.35 | Стандартный |

## 9. Преимущества и недостатки

### Flutter

**Преимущества:**
- Превосходная производительность анимаций и консистентность UI (собственный движок рендеринга)
- Impeller устраняет shader jank; стабильные 60/120 FPS
- Истинная мультиплатформенность: мобильные, веб, десктоп, embedded из одной кодовой базы
- Быстрорастущая доля рынка (46%)
- Hot reload для веба стал стабильным
- Идеален для визуально насыщенных, дизайн-ориентированных приложений

**Недостатки:**
- Dart — нишевый язык (соотношение JS:Dart разработчиков ~20:1)
- Меньшая экосистема пакетов (40-50K vs 1M+)
- Больший размер бинарных файлов приложений
- Веб-поддержка не идеальна для контентных/SEO-ориентированных сайтов
- Сложнее нанимать Dart-разработчиков

### React Native

**Преимущества:**
- Использует JavaScript/TypeScript — самую популярную экосистему языков программирования
- Проще найм: значительно больший пул разработчиков
- Огромная npm-экосистема (1M+ пакетов)
- New Architecture обеспечивает почти нативную производительность
- Мощная корпоративная поддержка: Meta, Microsoft, Shopify
- Expo значительно упрощает процесс разработки
- Переиспользование кода с веб-приложениями на React (30-40% экономия)
- 30-40% экономия по сравнению с отдельными нативными командами

**Недостатки:**
- Десктоп-поддержка менее зрелая (community-maintained через Microsoft)
- Сложные анимации всё ещё уступают Flutter
- UI через нативные компоненты = небольшая несогласованность между платформами
- Миграция на New Architecture может быть сложной для legacy-кодовых баз
- Удаление bridge может сломать старые сторонние нативные модули

## 10. Вывод и рекомендации

### Когда выбирать Flutter

- **Визуально насыщенные приложения:** кастомный UI, сложные анимации, уникальный дизайн
- **Мультиплатформенность из одной кодовой базы:** мобильные + десктоп + веб + embedded
- **Стартапы с нуля:** нет привязки к существующему JS-стеку
- **Финтех и автомобильная индустрия:** тренд на Flutter в этих отраслях
- **Консистентность UI:** одинаковый вид на всех платформах критически важен

### Когда выбирать React Native

- **Команда с опытом в React/JavaScript:** минимальная кривая обучения
- **Крупные проекты с большой командой:** проще масштабировать найм
- **Переиспользование кода с вебом:** значительная экономия для компаний с React-вебом
- **Нативный look & feel:** приложение должно выглядеть как «родное» для каждой платформы
- **Быстрый MVP:** Expo позволяет максимально быстро запустить продукт

### Общий вердикт

В 2026 году **оба фреймворка находятся на пике зрелости**. Flutter лидирует по производительности, мультиплатформенности и визуальному качеству. React Native лидирует по размеру экосистемы, доступности разработчиков и интеграции с веб-стеком.

Выбор зависит не от того, «какой фреймворк лучше», а от конкретного контекста проекта: команды, целевых платформ, типа приложения и бизнес-требований.

Стоит также учитывать растущую роль **Kotlin Multiplatform** (рост с 7% до 23% за 18 месяцев) — как третьего конкурента, особенно для проектов с фокусом на общую бизнес-логику при сохранении полностью нативного UI.
